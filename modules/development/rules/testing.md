# Правила тестирования

## Базовая политика

- Только JUnit 5 (`org.junit.jupiter`). Импорты JUnit 4 блокируются hook-ом.
- Каждый измененный production-класс должен иметь тестовый класс в зеркальном пути source set (`src/main/java/...Foo.java` -> `src/test/java/...FooTest.java`).
- Тесты пишутся по правилам `reference/junit-rules.md`, моки - по `reference/mocking.md`, проверки - по `reference/assertions.md`.

## Детерминированные проверки hook-ов

Блокирующие:

- импорты JUnit 4 (`org.junit.Test`, `junit.framework`);
- `@Disabled` / `@Ignore` без указания причины;
- ленивые имена методов вида `test1`, `test2`;
- тестовый файл с `@Test`, но без единого assert/verify.

Предупреждающие:

- отсутствие тестового файла для измененного production-класса (становится блокирующим на этапе Stop);
- файл длиннее 500 строк.

## Запуск тестов

Если в проекте обнаружен Gradle или Maven (предпочтительно через wrapper), агент `dev-test` запускает тесты сам. Если инструменты сборки недоступны, агент явно фиксирует это ограничение в результате.
