# Формат плана

План задачи хранится в `docs/plans/<task-slug>.md` и создается из шаблона `docs/templates/plan-template.md`.

## Обязательные секции

Hook `plan_gate` детерминированно проверяет наличие и непустоту секций (заголовки `##`, допускаются английские и русские варианты):

| Секция | Заголовок | Требование к содержимому |
|---|---|---|
| Цель | `Goal` / `Цель` | непустой абзац |
| Объем | `Scope` / `Объем` | что входит и что не входит |
| Шаги | `Steps` / `Шаги` | нумерованный список, не менее 2 пунктов |
| Затрагиваемые файлы | `Affected files` / `Затрагиваемые файлы` | не менее одного пути к файлу |
| Проверка | `Verification` / `Проверка` | упоминание тестов либо явное `no-tests: <причина>` |
| Риски | `Risks` / `Риски` | не менее одного пункта |

## Статус утверждения

- Шаблон создается со строкой `Status: draft`.
- После явного подтверждения пользователя статус меняется на `Status: approved` (допустимо `Статус: утвержден`).
- Редактирование `src/` без утвержденного плана вызывает запрос подтверждения от hook-а.

## Дополнительные правила

- В плане не должно оставаться маркеров `TODO`, `TBD`, `FIXME`.
- Строка `build-check: on` в плане включает запуск реальной сборки и тестов в Stop-hook.
- Строка `no-tests: <причина>` ослабляет требование тестов до предупреждения.
