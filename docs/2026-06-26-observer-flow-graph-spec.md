# Observer Flow-Graph — расширяемое представление флоу

> **Назначение:** самодостаточная спецификация расширения observer (WI-14/15):
> блок стадий из плоского графа становится информативным «пультом» — что
> происходит сейчас, как выглядит флоу по фазам, какие документы созданы, как
> проходят гейты.
>
> **Целевое место в репо:** `docs/2026-06-26-observer-flow-graph-spec.md`
> **Расширяет:** WI-14/15 (`observer.py`/`observer.html`), опирается на WI-13
> (`projection`, `_stage`), WI-11 (verdict), WI-20 (intake), WI-7/8 (contract/scope),
> WI-22 (`stages.json`).
> **Статус:** design / к реализации (ревью команды — на горячую, не пред-гейт).
> **Визуальный референс:** `.superpowers/brainstorm/.../with-doc-index.html` (v7-итог).
> **Язык:** ru-проза; технические идентификаторы — English/ASCII.

---

## 1. Проблема

Текущий блок стадий observer'а — плоский ряд из 4 узлов (`stages.json`:
intake/contract/plan/delivery). Он не отвечает на ключевые вопросы инженера:
**что агент делает сейчас? как выглядит флоу целиком? какие документы уже созданы
и где они? как проходят гейты?** Фаза implement не видна (нет узла), OpenSpec-доки
не показаны, ссылок на документы нет.

## 2. Что это

Расширение того же **read-only read-model** (P5): observer читает журнал +
артефакты и рендерит богатую проекцию. Без записи в управляющее состояние. Новое:
один серверный **read-only doc-route** (отдаёт содержимое файла из flow-дерева для
превью) — единственная новая серверная поверхность, под строгим скоупом.

## 3. Согласованный дизайн (5 зон, сверху вниз)

1. **«Сейчас»** — живая строка: текущая фаза, текущий под-шаг (напр. «implement —
   правка CardService.kt»), последнее решение гейта, остаток бюджета.
2. **Ряд фаз** — `intake → contract → plan → implement → delivery`. **plan и
   implement — отдельные фазы.** `implement` — **синтетическая** фаза (в
   `stages.json` её нет): observer вставляет её между plan и delivery, выводя из
   активности code-edit гейтов (как уже сделанный `implement`-индикатор). verify
   **остаётся внутри delivery** (не пятый узел) — в живой карте verify-стадии нет.
3. **Карточки фаз** — у каждой фазы своя карточка со своими шагами/документами:
   - `intake` — `intake.json` (есть/approved);
   - `contract` — `contract.json` (есть/approved/заморожен);
   - `plan` — OpenSpec-шаги: `proposal.md`, `specs/<cap>/spec.md`, `tasks.md`
     (+ прогресс N/M из чеклиста), `design.md` (опционально);
   - `implement` — технические шаги из журнала: правки (файл + гейт + решение),
     сводка «N правок · K ask · последняя Xс», оверфлоу «+N»;
   - `delivery` — verify (`verdict.json`), validate (`validate_development_output`:
     diff⊆scope · тесты · manifest), `pr-summary.md`.
4. **Полоса гейтов** — «прохождение гейтов»: каждый гейт окрашен по **последнему**
   решению (allow=зелёный / ask=жёлтый / block=красный / не запускался=серый).
   **Без счётчиков** (счётчики block/ask/allow живут в `vital signs`, не дублируем).
5. **Индекс документов** — сворачиваемый список всех документов флоу, разбит на два
   семейства, каждый файл со статусом и кликом → превью.

**Doc-превью:** клик по любому файлу открывает его содержимое read-only в
**модальном оверлее** (моноширинный, скролл; Esc/клик-вне закрывает).

## 4. Два семейства документов (важно — не путать)

| Семейство | Каталог | Файлы |
|---|---|---|
| **OpenSpec-change** | `openspec/changes/<slug>/` | `proposal.md`, `specs/<cap>/spec.md`, `tasks.md`, `design.md` (опц.) |
| **Артефакты флоу** | `docs/development/<slug>/` | `intake.json`, `contract.json`, `verdict.json`, `pr-summary.md` |

`pr-summary.md` — артефакт **delivery**, НЕ OpenSpec. Документы появляются
постепенно; `design.md` опционален. Чего нет на диске → статус «☐ ещё нет» (не
ошибка), появляется вживую при следующем snapshot.

## 5. Модель данных (добавления в enriched Snapshot)

Backend (`enrich`) добавляет, фронт собирает вид. Всё **read-only**.

```python
"documents": [        # плоский список; фронт группирует по фазе для карточек И индекса
  {"family":"openspec|flow", "phase":"plan", "path":"openspec/changes/card/tasks.md",
   "label":"tasks.md", "state":"present|approved|frozen|pending|optional",
   "progress":"2/5"|null}            # progress только для tasks.md (парс чеклиста)
],
"gates": [            # последнее решение по каждому гейту, в порядке pipeline
  {"gate":"git_guard", "decision":"allow|ask|block", "ts":"..."},
  {"gate":"build", "decision":null}   # null = не запускался
],
"implement": {        # РАСШИРЯЕТСЯ (поле уже есть): + steps
  "active":true, "edits":5, "last_sec":8,
  "steps":[{"file":"src/cards/CardService.kt", "gate":"gate_lint",
            "decision":"allow", "ts":"..."}]   # последние code-edit решения
}
```

- **Фаза-ряд** фронт выводит из `stage.stages` (intake/contract/plan/delivery) +
  вставляет `implement` из поля `implement` (между plan и delivery).
- **`documents`**: backend стат-ит известные пути под `<slug>` (+ glob
  `specs/*/spec.md`), читает статусы (approval-маркеры для approved/frozen; парс
  `tasks.md` для progress). Пути — относительные от root, нормализованные.
- **`gates`**: группировка `decisions` по полю `gate`, берём последнее по `ts`;
  упорядочиваем по фиксированному списку pipeline (`GATE_ORDER`).
- **`implement.steps`**: последние решения code-edit гейтов с резолвом файла из
  reason/tool (best-effort; если файл не извлекается — gate-имя).

## 6. Doc-route (новая серверная поверхность — банковский скоуп)

```
GET /doc?path=<rel>   →  200 text/plain; charset=utf-8   (содержимое файла)
                          400 — путь вне allowlist / содержит '..' / абсолютный
                          404 — нет файла
```

**Безопасность (обязательно):**
- `path` — относительный от root. Канонизировать (`os.path.normpath`), привести к
  `/`-форме, отклонить если: абсолютный, содержит `..`-сегмент после нормализации,
  или **не начинается** с одного из allowlist-префиксов: `docs/development/`,
  `openspec/changes/`. Никаких других каталогов в v1 (исходники src/ — **отложено**,
  шире поверхность).
- Резолвить `os.path.join(root, path)`, затем `os.path.realpath`, и проверить, что
  результат **внутри** `realpath(root)` (защита от симлинков). Иначе 400.
- Отдавать как `text/plain` (никакого исполнения/HTML-интерпретации), `utf-8`,
  **size-cap** (напр. 512 КБ; больше — усечь с пометкой). Только чтение.
- Read-only инвариант сохраняется: маршрут ничего не пишет.

## 7. Фронт (observer.html) — поведение

- Рендер 5 зон из enriched Snapshot. Фазовый ряд + карточки + полоса гейтов + индекс.
- **Раскрытие:** карточки фаз и индекс документов — сворачиваемые; состояние
  раскрытия в `localStorage` (как тема). Дефолт: карточка текущей фазы раскрыта.
- **Doc-превью:** клик по файлу → `fetch('/doc?path=...')` → модалка с содержимым
  (`<pre>`, экранировано через `esc`). Ошибка → «не удалось открыть» в модалке.
- **Темы** mint/ultra-pink, «?»-тултипы — как сейчас, распространить на новые блоки.
- **Zero external assets** сохраняется; всё инлайн.
- **fail-open:** нет документов/гейтов/implement → блоки «нет данных»/скрыты, не падать.

## 8. Границы (что НЕ делает в v1)

- **Не** превью исходников (src/*) — только flow-доки (`docs/development`,
  `openspec/changes`). Имена файлов в карточке implement — текст, не ссылка.
- **Не** редактирование/approve из вьюшки (read-only, P6).
- **Не** меняет `stages.json`/enforcement — `implement`/`verify` остаются
  визуально-производными, не новыми стадиями.
- VueFlow/граф-движок — по-прежнему отложен; рисуем flex/SVG.

## 9. Read-only / банковская поверхность

- Всё чтение; единственная новая поверхность — `/doc` (строго scoped, realpath-guard,
  text/plain, size-cap, reject `..`/absolute/outside-root). Bind по-прежнему
  `127.0.0.1`, ноль внешних ассетов, ноль записи.
- `/doc` наследует чувствительность журнала/доков (пути/диффы) — но не расширяет её
  за пределы того, что уже видно в swimlanes; отметить в модели угроз.

## 10. Acceptance criteria

- [ ] enriched Snapshot несёт `documents` (оба семейства, статусы, tasks-прогресс),
      `gates` (последнее решение/гейт, в порядке pipeline), `implement.steps`.
- [ ] Фронт рендерит 5 зон; фаза `implement` показана как синтетическая между plan
      и delivery; verify — внутри delivery-карточки.
- [ ] Полоса гейтов окрашивает гейты по последнему решению, без счётчиков.
- [ ] Индекс документов — сворачиваемый, два семейства, статусы и «☐ ещё нет/опц.».
- [ ] `/doc` отдаёт flow-док read-only; **отклоняет** `..`, абсолютные пути, пути вне
      allowlist, симлинк-выходы за root (юнит-тесты на каждый отказ).
- [ ] Doc-превью: клик → модалка с содержимым; Esc/клик-вне закрывает.
- [ ] Read-only доказан tree-hash'ем (включая запрос к `/doc`); bind 127.0.0.1;
      zero external assets — тесты держатся.
- [ ] Раскрытие карточек/индекса в `localStorage`; темы/тултипы работают на новых
      блоках.
- [ ] Юнит-тесты: `documents`/`gates`/`implement.steps`/tasks-прогресс на фикстурах;
      `/doc` security-battery; SSE/snapshot по-прежнему валидны.
- [ ] Live-приёмка на реальном стенде: открыть, увидеть фазы/карточки/доки/гейты,
      кликнуть документ → превью реального файла.

## 11. Открытые вопросы

- Doc-превью: модалка (выбрано) vs боковая панель — подтвердить на горячую.
- Полоса гейтов: фиксированный `GATE_ORDER` или порядок появления в журнале?
- implement.steps: глубина (сколько последних правок до оверфлоу)?
- Превью исходников (src/) — когда снимать ограничение (нужен скоуп по contract.scope)?
- Ветка: стек на `feature/observer-ux-text` (текст+implement, не влита) — да.
